## 二、所有人必看：格式规范

### 2.1 文件格式必须是 JSON，不是 Python

当前所有数据文件都是 `.py` 格式（Python 语法），需要全部转成 `.json`。区别如下：

```
❌ Python 格式（当前）：
- 单引号：'name'
- True / False
- # 注释
- 末尾多余逗号：["a", "b",]

✅ JSON 格式（目标）：
- 双引号："name"
- true / false
- 不允许注释
- 不允许末尾逗号：["a", "b"]
```

**怎么转？最简单的方法：**

1. 把你的 .py 文件内容复制到 ChatGPT / Claude
2. 说"帮我把这个 Python 字典转成合法的 JSON 格式，去掉注释和末尾多余逗号"
3. 把输出结果保存为 .json 文件
4. 用 https://jsonlint.com/ 验证一下格式是否正确

### 2.2 文件统一为纯数组格式

所有数据文件的最外层统一用数组 `[...]`，不要包外层对象。

```
❌ 不要这样：
{"city": "北京", "restaurants": [...]}
{"city": "北京", "gyms": [...]}

✅ 要这样：
[
  {"id": "r001", "name": "...", "city": "北京", ...},
  {"id": "r002", "name": "...", "city": "上海", ...}
]
```

原因：每条数据自带 city 字段，不需要外层再包一层。这样北京和上海的数据可以放在同一个文件里。

### 4.6 gyms.json（Skill 2 · 健身房数据）

**当前状态：** 5 条，缺少关键字段。

**需要做的事：**

| 优先级 | 任务 | 工作量 |
|--------|------|--------|
| P0 | 转 JSON 格式 | 5 分钟 |
| P0 | 给每条数据补 5 个关键字段 | 30 分钟 |
| P1 | 补 2-3 条上海健身房 | 20 分钟 |

**需要补的 5 个字段（每条都要加）：**

```json
{
  "...(现有字段保留)...": "...",

  "price_single": "29元/次",
  // 单次价格。如果不支持单次，写"无单次，最低月卡XX元"

  "equipment": ["有氧区", "固定器械", "自由力量区", "深蹲架x3", "史密斯机x1", "龙门架x1"],
  // 核心器械清单。这是用户问"练背去哪个健身房"时，管家判断器械是否匹配的关键依据
  // 重点标注：深蹲架数量、是否有自由力量区、是否有龙门架

  "suitable_for": ["力量训练", "有氧", "团课"],
  // 适合的训练类型

  "peak_hours": "工作日 18:00-20:00 人较多",
  // 高峰时段描述

  "cons": "自由力量区偏小，高峰期深蹲架需排队"
  // 已知短板。真实客观地写一个。可以参考大众点评差评中高频出现的问题
}
```

**怎么查这些信息：**
- 打开大众点评，搜索该健身房
- 看店铺详情页的"设施服务"标签，能看到器械照片和设施列表
- 看中差评（2-3 星），提炼出高频吐槽点作为 cons
- price_single 看美团/大众点评的团购价格
- peak_hours 看评论中提到的高峰情况，或用常识判断（工作日晚间通常最挤）

**上海健身房采集：**
- 在大众点评搜"陆家嘴 健身房"或"静安 健身房"
- 选 2-3 家，覆盖：1 家乐刻/超级猩猩（按次付费，出差友好）、1 家综合健身房、1 家酒店健身房（可选）
- 出差用户最关注"是否支持单次"，这个字段一定要填准

### 4.8 classes.json（Skill 2 · 团课/私教）

**当前状态：** 5 种团课 + 4 类私教，内容全面但缺少场馆关联。

**需要做的事：**

| 优先级 | 任务 | 工作量 |
|--------|------|--------|
| P0 | 转 JSON 格式 | 5 分钟 |
| P0 | 给团课绑定具体场馆 | 20 分钟 |

**怎么绑定场馆：**

现在的团课是"HIIT 高强度间歇训练"这种通用课程描述，管家不知道推荐用户去哪上。需要变成"XX 健身房的 HIIT 课"这种具体的、可推荐的课程。

改法：把现有 5 种团课"复制展开"成 8-10 条具体课程，每条绑定一个场馆。示例：

```json
// ❌ 当前（通用描述，没有场馆）：
{
  "id": "class_001",
  "name": "HIIT高强度间歇训练",
  "price_range": "¥50-120/次"
}

// ✅ 改成（绑定具体场馆）：
{
  "id": "class_001",
  "name": "HIIT 燃脂训练",
  "class_type": "HIIT",
  "venue_name": "乐刻健身(丽泽16号店)",
  "venue_district": "丰台区",
  "price": "59元/次",
  "duration": "45分钟",
  "level": "中等",
  "schedule": "每周一三五 19:00",
  "coach": "教练小K",
  "description": "高强度间歇，45分钟燃脂400大卡，适合有一定基础的人",
  "rating": 4.5
}
```

**操作方法：**
- 拿现有 5 种团课类型 × 2-3 个场馆 = 10-15 条具体课程
- 场馆从 gyms.json 里已有的健身房中选
- 教练名字随意编一个昵称
- schedule（上课时间）合理安排，工作日晚上和周末为主
- 价格参考美团上对应品牌的团课价格

私教部分同理，但优先级低一些，可以先不改。Demo 时私教场景不太可能是重点。
